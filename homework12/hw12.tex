\subsection{Linux 密碼儲存方式}
在目前 Linux 系統當中，跟密碼有關的有兩個檔案：\texttt{/etc/passwd} 和 \texttt{/etc/shadow}。這兩個檔案的差異在於：
\begin{itemize}
  \item \texttt{/etc/passwd}：儲存使用者帳號、UID、GID、使用者全名、家目錄和 shell 等資訊，但密碼欄位通常只包含一個特殊字元（如 \texttt{x} 或 \texttt{*}），表示密碼已被儲存在 \texttt{/etc/shadow} 中。
  \item \texttt{/etc/shadow}：專門用來儲存使用者的密碼資訊，包括加密後的密碼、密碼過期時間等。這個檔案只有 root 使用者可以讀取。
\end{itemize}
而在 \texttt{/etc/shadow} 中，每一行由多格經由冒號分隔的欄位組成。在第二欄當中，存的是加密後的密碼。這個欄位的格式通常是 
\texttt{\$<hash type>\$<salt>\$<hashed password>}。


% Problem 1(b)
\subsection{passwd 原理}
\texttt{passwd} 這個指令有一個特殊的權限叫做 \texttt{SUID}（Set User ID）。透過查看 \texttt{/usr/bin/passwd} 的權限，可以發現它的權限是 \texttt{rwsr-xr-x}，其中的 \texttt{s} 表示這個檔案具有 SUID 權限。並且，這個檔案是被 root 所擁有。當使用者執行這個指令時，因為 SUID 的緣故，系統會暫時將使用者的權限提升到該檔案擁有者的權限，也就是 root，因此這樣就可以修改 \texttt{/etc/shadow} 檔案中的密碼。

% Problem 1(c)
\subsection{登入 log}
登入的 log 記錄通常位於 \texttt{/var/log/auth.log} 檔案當中。這個 log 檔案會記載所有嘗試的登入事件，包含 \texttt{sudo}、\texttt{su}、SSH 登入等。每一行通常包含時間戳、動作、使用者名稱、來源 IP 等資訊。

\subsection{防範密碼破解}
\begin{itemize}
  \item 第一個防範方式是使用 \texttt{fail2ban} 這類軟體。\texttt{fail2ban} 會監控登入失敗的事件，並且在檢測到可能是惡意登錄的情況下，透過修改防火牆規則來封鎖該 IP 地址。這樣可以有效地防止暴力破解攻擊。
  \item 第二個防範方式較為簡單暴力，就是乾脆不要用密碼來登入，只允許使用 SSH 金鑰的方式來登入就好。這種方法的設置方式是透過在 \texttt{/etc/ssh/sshd\_config} 中，將 \texttt{PasswordAuthentication} 設為 \texttt{no}。這樣就會禁止使用密碼登入，只允許使用 SSH 金鑰登入。
\end{itemize}

\subsection{hide.py 運作原理}
\texttt{hide.py} 是以\texttt{LSB Steganography} 的方式將訊息隱藏在圖片中，方法如下：
\begin{enumerate}
  \item 首先，程式使用 Python 的 \texttt{PIL} 套件來將圖片每一個像素的 RGB 值讀取出來，並且存成一個陣列。
  \item 將訊息轉換成 8 位元的二進位制格式。
  \item 每一個字元的訊息會經由三個像素點的 RGB 值，總共 9 個位元來儲存。他的儲存方式是將每個像素的 RGB 值的最後一個位元（LSB） 改成訊息的二進位制位元。
  \item 最後再將圖片的像素值寫回到圖片中，並且儲存成新的圖片檔案。
\end{enumerate}

\subsection{圖中的訊息}
這題的破解方式非常簡單，就按照 \texttt{hide.py} 程式碼的邏輯反向操作就好。我撰寫了 \texttt{seek.py} 這個程式來讀取圖片中的像素值，並且將每個像素的 RGB 值的最後一個位元（LSB）組合起來，然後轉換成字元。最後就可以得到原本被隱藏在圖片中的訊息。
